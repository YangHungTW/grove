# Architecture — changes syntax highlight + open-in-IDE

Generated: 2026-06-17T01:39:45Z

## New modules
- `src/renderer/diffHighlight.ts`
  - `extToLang(path): string | undefined` — extension → highlight.js language id
    (only return a language hljs actually registers; else undefined).
  - `highlightDiffLine(text, lang): string` — if `lang` && `hljs.getLanguage(lang)`,
    return `hljs.highlight(text, { language: lang, ignoreIllegal: true }).value`
    (self-escaping); otherwise return `escapeHtml(text)`. The ONLY producer of the
    HTML injected via `dangerouslySetInnerHTML`.
- `src/core/ideLaunch.ts` (Electron-free, unit-tested)
  - `interface IdeConfig { command: string; terminal: boolean }`
  - `canOpenInIde(s: { ide?: IdeConfig }): boolean` — `!!command.trim()`.
  - `TERMINAL_EDITORS` + `isTerminalEditor(cfg)` — explicit flag OR known TUI binary.
  - `buildIdeOpenAction(cfg, filePath, ctx) : IdeOpenAction`
    - terminal → `{ mode:'session', request: CreateSessionRequest(kind:'shell',
      bootstrap: `${cmd} ${shellQuote(filePath)}\r`) }`
    - GUI → `{ mode:'exec', command: cmd, filePath }`

## Changed contracts
- `CreateSessionRequest` gains `bootstrap?: string` (ipc.ts). `launchSpecFor` shell
  branch returns it; `createSession` already forwards `spec.bootstrap` to PtySession.
  No new attack surface: a shell pane already executes whatever is typed via
  `sessionInput`, so a bootstrap is equivalent. The IDE bootstrap is built in **main**
  from the trusted `ide` setting; the renderer only supplies `filePath` (shell-quoted).
- New IPC `ide:open`: `ideOpen(filePath, ctx): Promise<SessionSnapshot | null>`
  (`ctx = { worktreeId, cwd, cols? }`). Main loads `ide` setting, throws if unset,
  calls `buildIdeOpenAction`; session-mode → `createSession()` (returns snapshot the
  renderer `placePane`s); exec-mode → `openInEditor()` (login-shell exec, honors
  `CCM_IDE_CMD` override), returns null.
- `AppSettings.ide?: IdeConfig` + `IDE_PRESETS` (VS Code/Cursor/Sublime/Zed/WebStorm
  = GUI; vim/nvim/helix = terminal). Default `ide: { command:'', terminal:false }`.

## UI
- `DiffPane`: wrap the file header in `.diff-file-headrow` (flex) so an
  `.diff-open-ide` icon button sits beside the (still-`<button>`) header without
  nesting. `disabled={!canOpenInIde(store.settings)}`; onClick →
  `store.openInIde(session.worktreeId, `${session.filePath}/${m.name}`)`. Per-line text
  → `dangerouslySetInnerHTML={{ __html: highlightDiffLine(l.text, lang) }}`, lang =
  `extToLang(m.name)`. `SplitHunk` takes a `lang` prop and does the same for `.diff-scell`.
- `store.openInIde(worktreeId, filePath)` — guard on `canOpenInIde`, call
  `window.api.ideOpen`, `placePane` the returned snapshot (vim) or no-op (GUI).
- `SettingsPanel`: "Editor" section — preset `<select>` (None / presets / Custom…),
  custom shows command input + "terminal editor" checkbox. Local `useState` tracks
  custom mode so an empty custom command still shows the inputs.
- `styles.css`: `.diff-text .hljs-*` token colours (dark-theme palette);
  `.diff-file-headrow` flex; `.diff-open-ide` icon button + `:disabled` dim.
