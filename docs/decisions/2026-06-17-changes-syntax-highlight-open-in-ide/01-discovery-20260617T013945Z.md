## Goal
Add per-language syntax highlighting to the Changes/diff view, and a per-file
"open whole file in IDE" action: disabled until an IDE is configured, GUI editors
launched via their CLI, terminal editors (vim) routed into an in-app shell.

## Decisions taken into architecture
- **highlight.js** (chosen in plan): synchronous, self-escaping. Empirically verified
  `hljs.highlight('<script>‚Ä¶', {language:'typescript'}).value` ‚Üí
  `&lt;script&gt;‚Ä¶&lt;/script&gt;`, no literal `<script>`. Guard unknown langs with
  `hljs.getLanguage(lang)` (falsy for bogus) and fall back to manual escape.
- Highlight **per diff-line** (plan scope): switch `.diff-text` / `.diff-scell` to
  `dangerouslySetInnerHTML` fed by `highlightDiffLine(text, lang)` (which is the only
  thing allowed to produce HTML ‚Äî it always escapes).
- New `ide?: { command, terminal }` setting; `IDE_PRESETS` carry the terminal flag;
  `buildIdeOpenAction` (core, pure, unit-tested) decides shell-session vs GUI-exec.
- vim path reuses the existing `shell` SessionKind + a new `bootstrap` on
  `CreateSessionRequest`; GUI path reuses the `runHook` login-shell exec.
